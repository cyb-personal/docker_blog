# 博客论坛 Docker 部署文档

## 一、概述

项目采用 Docker Compose 一键部署，共 **8 个容器**：

| 容器 | 镜像 | 端口 | 说明 |
|------|------|------|------|
| blog-mysql | mysql:8.0 | 3307 | 数据库 |
| blog-redis | redis:7-alpine | 6380 | 缓存 |
| blog-user | 自构建 | 8081 | 用户服务 |
| blog-post | 自构建 | 8082 | 帖子服务 |
| blog-comment | 自构建 | 8083 | 评论服务 |
| blog-like | 自构建 | 8084 | 点赞服务 |
| blog-gateway | 自构建 | 8080 | 网关 |
| blog-frontend | 自构建 | 3000 | 前端 Nginx |

后端服务不需要分别手动构建，Docker Compose 会自动依次编译打包每个服务。

---

## 二、前置条件（Ubuntu 服务器）

```bash
# 1. 安装 Docker
curl -fsSL https://get.docker.com | bash

# 2. 安装 Docker Compose v2（Docker 24+ 自带）
sudo apt-get install docker-compose-plugin -y

# 3. 验证安装
docker --version
docker compose version

# 4. 将当前用户加入 docker 组（避免每次 sudo）
sudo usermod -aG docker $USER
# 退出 SSH 重新登录生效
```

## 三、本地准备（Windows 开发机）

```powershell
# 1. 删除本地编译缓存（这些由 Docker 构建时自动生成，不用上传）
Remove-Item -Path "D:\blog-forum\backend\user-service\target"     -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item -Path "D:\blog-forum\backend\post-service\target"     -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item -Path "D:\blog-forum\backend\comment-service\target"  -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item -Path "D:\blog-forum\backend\like-service\target"     -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item -Path "D:\blog-forum\backend\gateway-service\target"  -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item -Path "D:\blog-forum\backend\common\target"           -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item -Path "D:\blog-forum\frontend\node_modules"           -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item -Path "D:\blog-forum\frontend\dist"                   -Recurse -Force -ErrorAction SilentlyContinue

# 2. 压缩项目（压缩后约 1~2MB，原项目包含缓存约 240MB）
# 右键 D:\blog-forum → 发送到压缩文件夹，或使用以下命令：
Compress-Archive -Path "D:\blog-forum" -DestinationPath "D:\blog-forum.zip"
```

## 四、上传到 Ubuntu


## 五、Ubuntu 上部署

```bash
# 1. 登录到 Ubuntu
ssh ubuntu@<服务器IP>

# 2. 解压
cd /home/ubuntu
unzip blog-forum.zip
cd blog-forum

# 3. 查看项目结构
ls -la
# 应该看到:  backend/  frontend/  docker-compose.yml  README.md  部署文档.md

# 4. 构建并启动所有服务
# 首次构建约 10~15 分钟（下载 Maven/Node 依赖 + 编译 Java + 打包前端）
# 后续启动只需几秒钟
docker compose up -d --build
```

### 构建过程详解

Docker Compose 会自动按依赖顺序执行以下步骤：

```
Step 1: 拉取基础镜像
  → mysql:8.0
  → redis:7-alpine
  → maven:3.8-openjdk-8-slim（后端构建阶段）
  → openjdk:8-jre-slim（后端运行阶段）
  → node:16-alpine（前端构建阶段）
  → nginx:alpine（前端运行阶段）

Step 2: 构建后端服务（5个）
  → common（依赖库，自动被引用）
  → user-service
  → post-service
  → comment-service
  → like-service
  → gateway-service
  每个服务编译过程：mvn clean package -DskipTests

Step 3: 构建前端
  → npm install
  → npm run build
  → nginx 配置

Step 4: 启动所有容器
  → MySQL → Redis → 4个业务服务 → 网关 → 前端
```

### 查看构建日志

```bash
# 实时查看所有服务的构建日志
docker compose up -d --build 2>&1

# 或者后台构建，再单独看日志
docker compose up -d --build
docker compose logs -f
```

## 八、日常运维命令

```bash
# 启动全部
docker compose up -d

# 停止全部（保留数据）
docker compose stop

# 停止并删除全部（保留数据卷）
docker compose down

# 停止并删除全部（包括数据卷，会清空数据库）
docker compose down -v

# 重启某个服务
docker compose restart user-service

# 重新构建并启动某个服务（代码修改后）
docker compose up -d --build user-service

# 查看实时日志
docker compose logs -f --tail=100

# 进入容器内部
docker exec -it blog-mysql mysql -uroot -p123456
docker exec -it blog-redis redis-cli
```

## 九、常见问题

### 1. 端口被占用

如果本地已有 MySQL(3306) 或 Redis(6379)，容器端口已改为 3307 和 6380，不会冲突。

如果某个端口被占用，修改 `docker-compose.yml` 中的 `ports` 映射即可。

### 2. 构建失败

```bash
# 查看具体错误
docker compose logs user-service

# 常见原因：网络问题导致 Maven/Node 依赖下载失败
# 解决：设置国内镜像源后再重试

# 对于 Maven（已在 pom.xml 中配置国内源可跳过）
# 对于 npm（在 frontend/Dockerfile 中已配置 registry=https://registry.npmmirror.com）

# 重新构建
docker compose up -d --build --no-deps user-service
```

### 3. 数据库数据持久化

MySQL 和 Redis 的数据存储在 Docker 命名卷中：
- `mysql-data` → `/var/lib/docker/volumes/blog-forum_mysql-data/`
- `redis-data` → `/var/lib/docker/volumes/blog-forum_redis-data/`

只要不执行 `docker compose down -v`，数据就不会丢失。

### 4. 代码修改后重新部署

```bash
# 修改代码后，只需重新构建对应的服务
# 例如修改了 user-service 代码：
docker compose up -d --build user-service

# 修改了前端代码：
docker compose up -d --build frontend
```

### 5. 如何查看数据库中的用户

```bash
docker exec -it blog-mysql mysql -uroot -p123456 -e "USE blog_forum; SELECT id, username, password, nickname FROM user;"
```
